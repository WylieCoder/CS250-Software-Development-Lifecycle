# CS-250: Software Development Lifecycle

## Overview

This repository contains coursework and project artifacts completed for CS-250: Software Development Lifecycle. The course focused on Agile and Scrum methodologies, sprint planning, user stories, and team collaboration throughout the software development process.

## Course Artifact

* Sprint Review and Retrospective

This artifact demonstrates the use of Agile practices to evaluate project progress, reflect on completed work, and identify opportunities for improvement.

## Reflection

### How do I interpret user needs and implement them into a program? How does creating user stories help with this?

I interpret user needs by focusing on the problem the user is trying to solve and the features they expect from the software. User stories help break larger requirements into smaller and more manageable tasks. They keep the development process focused on the end user and help ensure that the finished product meets user expectations.

### How do I approach developing programs? What Agile processes do I hope to incorporate into my future development work?

I approach software development by dividing projects into smaller pieces and completing work in stages. Agile practices such as sprint planning, sprint reviews, and retrospectives help organize development and allow teams to adapt to changing requirements. In future projects, I plan to continue using iterative development, user stories, and regular feedback to improve both the development process and the final product.

### What does it mean to be a good team member in software development?

Being a good team member means communicating effectively, completing assigned tasks, accepting feedback, and supporting other members of the team. Software development is a collaborative process, and successful teams depend on trust, accountability, and a willingness to adapt when challenges arise.
